## 2026-04-14 部署问题记录

### 问题 1：云开发环境配置错误

**错误信息：**
```
Error: project.config.json: cloudbaseRoot field needs to be string
```

**原因：**
`cloudbaseRoot` 字段值应该是字符串类型，但配置成了对象。

**解决方案：**
将 `project.config.json` 中的配置从：
```json
"cloudbaseRoot": {
  "envId": "cloud1-5gffqq3ue6220039"
}
```
改为：
```json
"cloudfunctionID": "cloud1-5gffqq3ue6220039"
```

### 问题 2：缺少 game.json 文件

**错误信息：**
```
Error: "game.json" file not found, or the file read failed
```

**原因：**
小游戏项目必须有 `game.json` 配置文件。

**解决方案：**
在根目录创建 `game.json` 文件：
```json
{
  "deviceOrientation": "portrait",
  "showStatusBar": false,
  "networkTimeout": {
    "connectSocket": 5000,
    "downloadFile": 5000,
    "request": 5000,
    "uploadFile": 5000
  }
}
```

### 问题 3：__tests__ 目录为保留目录

**错误信息：**
```
Error: 非法的文件，错误信息：以"__"开头和结尾的目录为保留目录 : /cloudfunctions/assets/__tests__/index.test.js
```

**原因：**
微信云开发平台将以 `__` 开头和结尾的目录视为保留目录，不允许上传。单元测试文件只能本地运行。

**解决方案：**
1. 删除所有云函数中的 `__tests__` 目录：
   ```bash
   rm -rf cloudfunctions/*/__tests__
   ```

2. 创建 `.wxignore` 文件排除测试文件：
   ```
   **/__tests__/**
   **/*.test.js
   ```

3. 单元测试只能在本地通过 `npm test` 运行，不能上传到云端。

---

### 问题 4：代码包大小超过限制

**错误信息：**
```
Error: 代码包大小超过限制，subpackage __FULL__ source size 74109KB exceed max limit 4096KB
```

**原因：**
- `node_modules` 目录被打包上传
- `library/`、`temp/` 等 Cocos Creator 生成目录过大
- 微信小游戏代码包限制为 4MB

**解决方案：**
1. 删除所有 `node_modules` 目录：
   ```bash
   rm -rf cloudfunctions/*/node_modules
   ```

2. 更新 `.wxignore` 文件，排除大型目录：
   ```
   # 忽略 node_modules
   cloudfunctions/*/node_modules/**
   cloudfunctions/**/node_modules/**

   # 忽略 Cocos Creator 生成的大型目录
   library/**
   temp/**
   .profiler/**
   .local/**

   # 忽略其他项目目录
   ../freelancework/**
   ../zshield_workspace/**
   ```

3. 部署云函数时勾选"上传时云端自动安装依赖"
